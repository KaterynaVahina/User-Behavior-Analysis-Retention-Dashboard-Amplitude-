# User Behavior Analysis & Retention Dashboard (Amplitude)

## Огляд проєкту
Провела комплексний аналіз поведінки нових користувачів мобільного додатка зі США. Проєкт фокусується на дослідженні воронки онбордингу, утриманні (Retention) та порівнянні активності між платформами iOS та Android.

## Технологічний стек
* **Amplitude:** аналіз поведінкових паттернів, побудова воронки (Funnel) та розрахунок Retention.
* **SQL:** сегментація користувачів, створення цільових когорт та агрегація даних.

## Бізнес-метрики (KPI)
* **Onboarding Funnel:** Welcome → User Sign Up → Main Landing Screen → Search Song/Video → Play Song/Video.
* **Retention Rate:** відсоток утримання користувачів від першого входу.
* **Time to Complete Onboarding:** швидкість адаптації нового користувача.
* **Platform Performance:** порівняння метрик iOS та Android.

## Аналіз та результати
* **Ефективність воронки:** 72.1% користувачів успішно доходять до цільової дії «Play Song/Video» в перший день. Повний шлях онбордингу завершують 16.2% когорти.
* **Показники Retention:** 30-денний Retention склав 37.29%. Аналіз за платформами виявив вищу стабільність iOS-користувачів (62.9% на 7-й день проти 62.66% у Android).
* **Швидкість адаптації:** Середній час проходження воронки онбордингу становить ~4 години.

## Ключові інсайти
* **UX-бар’єри:** Виявлено критичну точку відтоку (32.1%) між етапами «Main Landing Screen» та «Search Song/Video». Це вказує на можливі труднощі у взаємодії з інтерфейсом пошуку.
* **Точка зацікавленості:** Найбільша концентрація активних користувачів (49.3% від загальної когорти) спостерігається на етапі «Search Song/Video», що робить цей екран ключовим для утримання.

## Рекомендації
* **UX-оптимізація:** Переглянути інтерфейс переходу до пошуку для зниження відтоку на 32%.
* **Технічний аудит Android:** Провести дослідження причин нижчого Retention на Android-пристроях та адаптувати онбординг.
* **Робота з фокусом:** Покращити функціонал екрана «Search Song/Video» для закріплення успіху найпопулярнішого етапу воронки.
